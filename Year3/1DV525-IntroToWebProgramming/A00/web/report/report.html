  1. Explain how the scope works in JavaScript, how hoisting affects it and how a closure works.

    The scope in JS is very similar to how the scope in Java works. For example, if the a variable is initialised inside of a If statement, the value of that variable will only be accessible inside this If statement and will throw Reference error if it is being called outside! For hoisting, all the functions or variables will be declared at the top of their scope. So if a variable is declared outside all the functions it will become global variables and will be available to use everywhere in the code! For Closures it is when two functions are bundled together where the inner function has access to the scope of the outer function. They are often used for object data privacy or event handlers.

  1. For each feature (F1, F2 and so on) you have implemented, write a 5-10 sentences text about it and let the teacher know how you solved it. This is essential. There is no grade if you fail to write about the features.

    F1: To be able to do a single page application, I have used ES6 Modules. For dragging the windows I have used EventListeners onmousedown, onmouseup and onmousemove. It allowed me to track where the user is dragging the window and therefore allows me to change the position of the window inside the PWD. To be able to solve the focus, I have just created a global variable everytime there is a click on the window I increment this number and sets the "zIndex" of this div which makes it get the focus.

    F2: For this assignment I only have around 20 commits. I always struggle to remember to commits everytime I finish a functionality. I have used JSDoc for every function and constructors of the code. For the ES modules, I have organised my code into 4 modules. One for the controller "app.js", one for the chat, one for the Memory Game and one for the Weather module. Concerning the npm test, I have fixed all the errors that test were telling me.

    F3: For the MemoryGame, I have decided to first show a menu t o be able to select which size of the game they want to use. To be able to choose the order of the picture I create an array filled with numbers where each number appears twice. Then I shuffle this array using the Fisher-Yates algorithm. The numbers represents the images name and this how i randomise them. To be able to show them on screen I use row and columns depending on the size. I store the image in the div value and whenever someone click on the div I will add the image to the div which will make it appear on screen.

    F4: For the Chat Application, the first time the user will open the app it will ask for an username. This username will be stored on the localStorage so that everytime he open the chat App again. The user will be already connected with this username. I use a webSocket to be able to connect to the server. I am using EventListeners such as onerror, onopen and onmessage to be able to show the messages on the screen as soon as they arrived or show it on the screen the user is connected. The user can also see at least 20 messages before the scrollbars appear.

    F5: For the Weather Application, I have used an API from openweathermap.org. It allowed me to fetch data depending on which city the user want to see. After sending the fetch request I received a JSON file back with a lot of weather information. I then only select the information I thought was important and put it on the window as content. And if the suer writes down a not existing location the API will let me know and I can tell the user that the city was not found or does not exists according to the API.
